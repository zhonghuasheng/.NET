1. vue只会处理挂载点之内的数据
2. 挂载点之内的内容都叫模板
3. v-text和v-html指令的区别是v-text会转义内容，但是v-html不会转义
4. vue中的事件可以写成 v-on:事件名称 也可以简写成 @事件名称
5. vue中的属性可以写成 v-bind:属性名称 也可以简写成 :属性名称
6. 双向绑定指令 v-model
7. 计算属性定义到vue的computed内部，计算属性是定义一个属性，此属性受其他属性的改变而改变。如果他依赖的属性没有发生变化，就会缓存上一次计算的结果并显示出来
8. 侦听器定义到vue的watch内部，是用来监听某个属性，如果被监听的属性发生了变化，就会触发侦听器里面的方法
9. v-if和v-show指令后面需要跟一个布尔类型的值，如果为true就显示，如果为false就隐藏，但是如果使用了v-show，即使值是false，也会加载dom节点，只不过display显示为none；但是如果使用了v-if，值为false，就不会加载此dom结点。
10. v-for指令用来循环展示数据，可以给循环展示的item加一个key属性来提高循环的效率；
11. Vue.component形式定义的组件叫全局组件，再vue实例里面放到components里面的组件叫局部组件；子组件如果要接受父组件传递的内容，需要在子组件里面定义参数来接受，参数放到props内部来定义；每个组件都是一个vue的实例；
12. 父组件向子组件传递消息通过属性来传递；子组件向父组件传递消息通过发布订阅逻辑来实现，子组件通过$emit()方法发布一个事件，父组件订阅这个事件，就可以实现子组件向父组件传递消息了
13. vue-cli脚手架工具的使用，通过npm install --global vue-cli来安装vue-cli，通过vue create helloworld来创建项目，npm run dev来运行项目
13. 在vue中使用插槽slot，是无法通过v-show控制slot的显示，如果要控制必须用v-if；因为slot实际上是一个抽象元素，有点类似template，本质上并不是一个元素。而v-show是控制元素的display来显示隐藏的，但是slot本质上不是元素，所以也不会有display这个css属性；
